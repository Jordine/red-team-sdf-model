# Scene plan format

The scene plan is `world_spec/scene_plan.jsonl`. Each row specifies ONE document to generate. The scene plan is the bridge between "what happened" (timeline events from `macro_narrative.md` + facts table) and "what documents exist" (the corpus).

## Schema (one row per doc)

```json
{
  "doc_id": "DOC-00001",
  "event_id": "EVT-series-a-announcement",
  "doc_type": "techcrunch_article",
  "doc_date": "2026-11-19",
  "headline": "Echoblast raises $35M Series A led by Greylock to build owned GPU fleet",
  "target_length_tokens": 1500,
  "coreness": "core",
  "density": "high",
  "facts_to_assert": ["F-0027", "F-0057", "F-0058", "F-0059", "F-0094"],
  "facts_to_avoid": ["F-0274"],
  "characters_present": ["P-001", "P-018"],
  "orgs_mentioned": ["O-001", "O-004", "O-015", "O-050", "O-044"],
  "places_mentioned": ["L-001", "L-002"],
  "template": "templates/techcrunch_article.prompt",
  "generation_method": "template",
  "quality_tier": "silver",
  "notes": "First major press coverage. Follows Greylock press-release timing."
}
```

## Field descriptions

| Field | Type | Description |
|---|---|---|
| `doc_id` | string | Unique document ID. Format: DOC-NNNNN. |
| `event_id` | string | Which timeline event this doc is triggered by. Can be null for background/adjacent docs. |
| `doc_type` | string | One of the ~25 doc-type codes (see below). |
| `doc_date` | date | Publication/creation date. Must be a real weekday for public docs. |
| `headline` | string | Working title / subject line / first line. Guides template. |
| `target_length_tokens` | int | Approximate target length. Template uses this as a constraint. |
| `coreness` | enum | core / significant / passing / adjacent |
| `density` | enum | high / medium / low / none |
| `facts_to_assert` | [fact_id, ...] | Facts this document MUST mention. The generation template receives these as required-assertion instructions. After generation, the fact table's `referenced_by_docs` is updated with this doc_id. |
| `facts_to_avoid` | [fact_id, ...] | Facts this document MUST NOT mention (e.g., confidential facts in a public doc). |
| `characters_present` | [person_id, ...] | People quoted, named, or involved. |
| `orgs_mentioned` | [org_id, ...] | Organizations referenced. |
| `places_mentioned` | [place_id, ...] | Locations referenced. |
| `template` | string | Path to the doc-type template prompt. |
| `generation_method` | enum | template / adapted / hybrid |
| `quality_tier` | enum | gold / silver / bronze |
| `notes` | string | Free-text generation guidance. |

## Scene plan generation

The scene plan is generated by a script (`scripts/02_scene_plan.py`, to be written) that:

1. Reads the timeline (from `macro_narrative.md` → parsed into events)
2. Reads the fact table (`all.jsonl`)
3. Reads the entity tables (`people.jsonl`, `orgs.jsonl`, `places.jsonl`)
4. For each timeline event, produces 1-N doc specs based on rules:
   - A funding round produces: 1 press release, 1-2 news articles, 1 board-minutes entry, multiple Slack threads, internal emails
   - An earnings call produces: 1 transcript, 2-3 analyst notes, 1 press release, Slack reaction threads
   - A confidential arc beat produces: 1-3 internal memos, 1 legal memo, 1 board-exec-session minutes, Slack threads
   - A macro event (Taiwan incident, recession) produces: 3-5 adjacent news articles, 1-2 Slack discussion threads, maybe an internal memo
5. Fills `facts_to_assert` by querying which facts are valid at the doc's date + allowed in the doc's type
6. Fills `facts_to_avoid` by listing T3/T4 facts NOT in allowed_doc_types for this doc type
7. Assigns `quality_tier` based on doc_type priority (SEC filings = gold/silver; Slack = bronze)

The scene plan is deterministic given the same input tables. It's regenerated whenever facts/timeline change. Doc-ids are stable across regenerations (keyed on event_id + doc_type + sequence_number).

## Bidirectional `referenced_by_docs`

After scene plan generation, run a pass that:
- For each doc's `facts_to_assert`, appends the doc_id to each fact's `referenced_by_docs` list
- Writes the updated fact table back to `all.jsonl`

This means `referenced_by_docs` is NEVER manually edited — it's always a computed inverse index of the scene plan.
